FAQ

  1. Will you support other download managers as Flashgot did?

    No. This is one of my leisure time project and I do not have enough time to invest on supporting other download managers. However, you have the freedom to fork it and adapt to other download managers.

  2. Why did you choose uGet over others?

    I find uGet as the only GUI download manager with all required features and native implementation. So in my point of view, uGet is the best Free and Open Source download manager out there for Linux users.

  3. Why does uGet Integration extension require so many permissions?

    Whatever permissions required are necessary for the functionality of the extension. There is no misuse of the permissions and you can check the code as it is an open source project.

  4. uGet Integration extension does not interrupt Google Drive

    Google Drive is purposely ignored due to a known problem (See the GitHub issue #2).

  5. uGet Integration extension does not interrupt streams?

    If you set the minimum file size to interrupt the download to -1, the extension will interrupt streams. However, uGet download manager may fail to download those files successfully.

  6. Why does the extension show Error: Unable to connect to the uget-chrome-wrapper?

    If the uget-chrome-wrapper is not installed or setup properly, you may get this error. If you have alreay installed all the required components, a system restart may resolve the issue. If not, please check the GitHub issue #20.

  7. The Firefox addon does not interrupt the download but download via right click context menu works

    If Firefox does not interrupt downloads but Download with uGet works, delete the handlers.json from the following path: ~/.mozilla/firefox/mwad0hks.default/handlers.json. You may have a different folder name instead of mwad0hks.default. Firefox Nightly users may have firefox-trunk folder instead of firefox.

  8. The extension does not interrupt downloads in Windows

    Windows users may have prblems with longer PATH variable. Please check the GitHub issue #43 and #45.
